#### DF-002

### Sumary

Benefits Dashboard - Login page - 405 - invalid credentials

### Type

BE

### Description

The login page for the Benefits Dashboard returns an HTTP 405 Method Not Allowed error when invalid credentials are submitted.
In this scenario, the expected behavior is to display an error message and allow the user to try again.
It appears that the `redirectURL` is missing, resulting in the following error: `_error: "net::ERR_HTTP_RESPONSE_CODE_FAILURE"`
When entering a valid username but an incorrect password, the application correctly displays an error message.
This inconsistency may expose a potential vulnerability, as attackers could use this behavior to determine which usernames are valid.

Example from HAR:

```
"redirectURL": "",
"headersSize": -1,
"bodySize": -1,
"_transferSize": 438,
"_error": "net::ERR_HTTP_RESPONSE_CODE_FAILURE",
"_fetchedViaServiceWorker": false
```

HAR: [loginPage.har](../data/loginPage.har)
Date: Sun, 29 Mar 2026 09:44:27 GMT

###### Steps to reproduce:

1. Go to login page
2. Enter invalid credentials and log in
3. HTTP error 405 Method Not Allowed

### Severity

High
