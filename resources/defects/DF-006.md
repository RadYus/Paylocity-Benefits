#### DF-006

### Sumary

Benefits Dashboard - Calculations - Incorrect Decimal Precision in Benefits Cost / Net Pay

### Type

BE

### Description

The backend calculations for Benefits Cost and Net Pay occasionally produce values that differ from mathematically correct results by a few decimal places.

While the FE rounds these values and hides the discrepancy, the backend still stores and returns imprecise numbers, which can cause cumulative errors in reporting or payroll‑related logic.

For exaple situation with 10 dependants:

#### Benefits Cost - Expected: 230.76923 / Received: 230.76926 / True Value: 230,7692307692308

#### Net Pay - Expected: 1769.2308 / Received: 1769.2307 / True Value: 1769,230769230769

POST/PUT API: https://wmxrwq14uc.execute-api.us-east-1.amazonaws.com/Prod/api/Employees

Calculation output: [calculationData.md](../data/calculationData.md)

Calculation function: [beTestData.js](../automation/be/beTestData.js)

###### Steps to reproduce:

1. Call POST/PUT API with (for exaple) 10 dependants
2. In respond body Benefits Cost and Net Pay doesn't equal calculations

### Severity

Medium-High
