#### DF-006

### Sumary

Benefits Dashboard - Calculations - Benefits Cost/Net Pay

### Type

BE

### Description

Calculations for "Benefits Cost" and "Net Pay" are sometimes off by decimal values.
It is so insignificant, that it won't be visible in Benefits Dashboard FE, but it could be problematic if all numbers are accounted for.

For exaple situation with 10 dependants:
Benefits Cost - Expected: 230.76923 / Received: 230.76926 / True Value: 230,7692307692308
Net Pay - Expected: 1769.2308 / Received: 1769.2307 / True Value: 1 769,230769230769

POST/PUT API: https://wmxrwq14uc.execute-api.us-east-1.amazonaws.com/Prod/api/Employees
Calculation output: [calculationData.md](../data/calculationData.md)
Calculation function: [beTestData.js](../automation/be/beTestData.js)

###### Steps to reproduce:

1. Call POST/PUT API with (for exaple) 10 dependants
2. In respond body Benefits Cost and Net Pay doesn't equal calculations
