#### DF-005

### Sumary

Benefits Dashboard - First/Last name - special characters

### Type

BE/FE

### Description

Users can submit employee names containing characters such as `/ , ] [ { }` or numbers.
This issue occurs:

- In the API (POST/PUT) — backend does not validate name fields.

- In the FE form — frontend does not block or sanitize invalid input.

As a result, invalid employee records can be created and displayed in the UI.

POST/PUT API: https://wmxrwq14uc.execute-api.us-east-1.amazonaws.com/Prod/api/Employees

###### Steps to reproduce BE:

1. Call POST/PUT API with special characters `/ , ] [ { }` or numbers in request body: "firstName" and "lastName"
2. In respond body, there are special characters in places of "firstName" and "lastName"
3. Employee is saved with special characters

###### Steps to reproduce FE:

1. Log in to FE Benefits Dashboard
2. On the "Benefits Dashboard" page, click the "Add Employee" button
3. Input Employee First/Last Name with special characters `/ , ] [ { }` or numbers
4. Input Dependents number between 0-32
5. Click the "Submit" button
6. Employee is saved with special characters

### Severity

High
